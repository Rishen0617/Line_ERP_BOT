## 系統簡介

本系統讓你直接在 **LINE 群組**中處理日常商務文件，不需要手動輸入 Excel：

```
你在 LINE 傳送圖片或訊息
        ↓
AI 自動辨識內容（Claude Vision）
        ↓
自動存入 Google Sheets（試算表）
        ↓
圖片同步備份到 Google Drive
        ↓
Dashboard 即時顯示收支概況
```

**支援功能一覽：**

| 功能 | 說明 |
|------|------|
| 圖片 OCR | 收據、發票、訂購單、出貨單 → 自動辨識填表 |
| 文字記帳 | 打幾個字就記錄收支 |
| 訂單管理 | 叫貨記錄自動建立，追蹤進度 |
| 物流查詢 | 貼上物流單號即可查詢 |
| 斜線指令 | `/月報`、`/查帳`、`/訂單`、`/查 單號` |
| Dashboard | 瀏覽器開啟即可查看圖表與明細 |

## 如何加入 LINE 群組

1. 請管理員將你加入已設定好的 LINE 群組
2. Bot 會自動監聽群組中所有訊息
3. **不需要加 Bot 為好友**，只要在群組內即可使用全部功能

> 💡 Bot 不會回應一般聊天，只處理符合格式的訊息

## 上傳單據（圖片辨識）

### 操作步驟

1. 在群組中點擊 **「+」→「相片」**（或直接拍照）
2. 選擇收據、發票、出貨單的照片
3. 傳送後，Bot 會在幾秒內回覆辨識結果

### 回覆範例

```
✅ 發票辨識完成

單號：AB-12345678
日期：2026-05-10
廠商：統一超商
品項：礦泉水 x10、飲料 x5
金額：NT$1,250
稅額：NT$60
信心度：高

📎 圖片已備份到 Drive
📊 已記錄到 Sheets（第 42 列）
```

### 信心度說明

| 信心度 | 意思 | 建議動作 |
|--------|------|----------|
| **高** | AI 確定辨識正確 | 直接使用 |
| **中** | 大部分正確，少數欄位不確定 | 建議到 Sheets 核對金額 |
| **低** | 圖片不清晰或格式特殊 | 請重拍或手動輸入 |

### 拍照建議

- 光線充足，避免反光
- 紙張攤平，不要摺疊
- 確保整張單據都在畫面內
- 文字最好清晰可見，不要模糊

### 支援的單據類型

| 類型 | 說明 |
|------|------|
| 發票 | 統一發票、電子發票 |
| 收據 | 一般購物收據 |
| 訂購單 | 向供應商的採購單 |
| 出貨單 / 送貨單 | 廠商送來的出貨憑單 |

## 文字記帳

不需要圖片，直接打字就能記帳。

### 格式

```
[類別關鍵字] [金額] [對象] [說明]
```

### 支出範例

```
支出 3200 王老吉食品 進貨礦泉水
付款 12500 新光纖維 布料
```

### 收入範例

```
收入 85000 客戶A公司 5月貨款
收款 5000 張老闆 還款
```

### 應付 / 應收範例

```
應付 28000 台塑石化 燃料費（月底付）
應收 15000 王大同 未收貨款
```

### Bot 回覆

```
✅ 已記帳
支出 NT$3,200
對象：王老吉食品
說明：支出 3200 王老吉食品 進貨礦泉水
已存入：流水帳 第 28 列
```

> 💡 **金額超過 NT$100,000** 時系統會自動發出警示通知管理員

## 叫貨與訂單管理

### 建立訂單

在群組中輸入含有叫貨關鍵字的訊息：

```
叫貨 王老吉 礦泉水 100箱
訂購 聯華食品 洋芋片 50箱
進貨 統一企業 泡麵 200件
採購 台灣菸酒 高粱酒 10箱
```

### Bot 回覆

```
✅ 訂單已建立（待確認）
供應商：王老吉
品項：叫貨 王老吉 礦泉水 100箱
已存入：訂單追蹤 第 15 列
```

### 訂單狀態

到 Google Sheets「訂單追蹤」工作表手動更新狀態：

| 狀態 | 說明 |
|------|------|
| 待確認 | 剛建立，尚未與廠商確認 |
| 已確認 | 廠商已接單 |
| 出貨中 | 廠商已出貨 |
| 已到貨 | 商品已到 |

## 物流查詢

直接在群組中貼上物流單號，系統會自動辨識快遞公司並提供查詢連結。

### 支援格式

| 快遞公司 | 單號格式 | 範例 |
|----------|----------|------|
| 黑貓宅急便 | 2 碼英文 + 10 碼數字 | `AB1234567890` |
| 7-11 交貨便 | 12 碼數字 | `123456789012` |
| 郵局包裹 | R/V + 9 碼數字 + TW | `R123456789TW` |

### Bot 回覆範例

```
📦 黑貓宅急便
單號：AB1234567890
🔗 查詢連結：https://...（點此查詢最新狀態）
```

## 查詢指令（斜線指令）

在群組中輸入以下指令：

### `/月報`

查詢本月收支摘要

```
/月報
/月報 2026-04    ← 指定月份
```

**回覆範例：**
```
📊 2026年5月 收支摘要
💚 收入：NT$185,000
❤️  支出：NT$142,500
📋 應收：NT$30,000
📋 應付：NT$18,000
─────────────
✅ 淨額：NT$42,500
```

---

### `/查帳`

查看最近 10 筆流水帳記錄

```
/查帳
```

---

### `/訂單`

查看最近 5 筆訂單狀態

```
/訂單
```

---

### `/查 單號`

用發票或單據號碼查詢是否已上傳

```
/查 AB-12345678
/查 TEST-20260510
```

## 控制台 Dashboard

用瀏覽器開啟即可查看圖表，無需安裝任何程式。

### 開啟方式

**本機（同一台電腦）：**
```
http://localhost:8001/dashboard
```

**透過 ngrok（其他設備、手機）：**
```
https://你的ngrok網址/dashboard
```

### 頁面內容

**上方 KPI 卡片（6 張）**
- 本月收入
- 本月支出
- 本月淨額（自動顯示紅/綠色）
- 應付帳款
- 收據總筆數
- 待處理訂單數

**近 6 個月收支長條圖**
- 綠色 = 收入　紅色 = 支出

**最近 30 筆收據明細**
- 含日期、廠商、金額、AI 信心度
- 圖片欄位可直接點擊開啟 Drive 備份

**最近 30 筆訂單追蹤**
- 含狀態標籤（顏色區分進度）

> 💡 點擊右上角「**重新整理**」可更新資料

## Google Sheets 說明

所有資料自動儲存在同一份 Google 試算表，共 4 個工作表：

### 工作表 1：收據記錄

圖片辨識結果自動寫入此表。

| 欄 | 說明 |
|----|------|
| A | 文件日期 |
| B | 上傳時間 |
| C | 上傳者 |
| D | 文件類型 |
| E | 廠商／客戶 |
| F | 品項摘要（前 3 項） |
| G | 金額 |
| H | 稅額 |
| I | 單號（唯一，重複不寫入） |
| J | Drive 圖片連結 |
| K | AI 信心度 |
| L | 群組 |
| M | 備註 |

### 工作表 2：記帳流水帳

文字記帳、圖片辨識後的財務記錄。

| 欄 | 說明 |
|----|------|
| A | 日期 |
| B | 類別（收入／支出／應付／應收） |
| C | 說明 |
| D | 金額 |
| E | 對象 |
| F | 關聯單號 |
| G | 記錄者 |

### 工作表 3：訂單追蹤

叫貨、訂購記錄。**狀態欄請手動更新。**

### 工作表 4：月報

公式自動計算，程式不寫入此表。可自行在 B1、C1 填入年份和月份查看。

## 常見問題

**Q：傳圖片後 Bot 沒有回覆？**
A：確認 Bot 伺服器是否還在執行（`http://localhost:8001/health` 應顯示 `{"ok":true}`）。若無回應，重新執行 `uvicorn app.main:app --port 8001`。

---

**Q：辨識結果的金額錯誤怎麼辦？**
A：直接到 Google Sheets「收據記錄」工作表修改對應列的金額即可。Sheets 中的資料隨時可以手動編輯。

---

**Q：同一張發票重複傳送？**
A：系統會比對單號（欄 I），**相同單號不會重複寫入**。若單號辨識失敗（留空）則視為新筆資料。

---

**Q：金額超過多少會發出警示？**
A：預設 **NT$100,000**，可在 `.env` 修改 `AMOUNT_ALERT_THRESHOLD` 設定值。

---

**Q：支援哪些圖片格式？**
A：JPG、PNG、HEIC（iPhone 拍攝）、PDF（單頁）皆可。建議直接用 LINE 拍照或選取相簿照片。

---

**Q：資料會保留多久？**
A：資料存在 Google Sheets 和 Drive，保留期限由 Google 帳號決定，不受本系統影響。

---

**Q：多個群組可以用同一個 Bot 嗎？**
A：可以。不同群組的資料會一起寫入同一份 Sheets，「群組」欄（L 欄）會標記來源。

---

**Q：Dashboard 的資料多久更新一次？**
A：每次點「重新整理」才會更新，資料直接從 Google Sheets 即時讀取。
