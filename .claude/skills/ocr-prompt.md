# OCR Prompt 調優技巧

## 常見問題與解法

### 問題 1：信心度總是回傳「低」
**原因：** 圖片解析度不足或模型 context 不夠清楚。

**解法：**
- 在 prompt 最前面加「[圖片品質良好，請盡力辨識]」
- 調整 `source_hint` 參數，例如 `"統一發票"` 幫助模型縮小範圍

### 問題 2：金額解析錯誤（如 12,500 解析成 12 或 500）
**原因：** 逗號被當作分隔符。

**解法：** 在 prompt 中強調：
```
金額格式：台灣使用千分位逗號（如 12,500 是一萬兩千五百元）。不要把逗號當小數點。
```

### 問題 3：民國年份解析錯誤
**原因：** 模型可能不熟悉民國曆。

**解法：** prompt 中加入：
```
台灣常用民國年：民國年 + 1911 = 西元年。
例：民國115年 = 2026年，民國113年 = 2024年。
```

### 問題 4：發票號碼格式辨識錯誤
**原因：** 統一發票格式特殊。

**解法：** 在 prompt 加入範例：
```
台灣統一發票格式：2碼大寫英文 + 連字號 + 8碼數字
例：AB-12345678、XY-98765432
```

### 問題 5：多品項漏失
**原因：** 圖片品項太多，模型截斷。

**解法：**
- 增加 `max_tokens`（目前 2048，可調至 4096）
- 在 prompt 強調：「所有品項都必須列出，不得省略」

## Prompt 版本測試方法

1. 把收據圖片 base64 後呼叫 `/admin/test-ocr`
2. 比較 `confidence` 和 `ocr_raw` 字段
3. 若多數欄位為 null，代表 prompt 描述不夠清晰

## A/B 測試建議

維護兩個 prompt 版本（`ocr_receipt_v1.txt` / `ocr_receipt_v2.txt`），
在 `claude_service.py` 用 flag 切換，收集 20 筆樣本後比較正確率。
