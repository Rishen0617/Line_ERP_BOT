## 系統架構

```
Windows 本機
├── F:\AI\line-sme-bot\      ← 本系統（FastAPI + Python）
│   ├── app\                 ← 主程式
│   ├── ERP bot.xlsx         ← 本地主要資料庫（優先寫入）
│   ├── .env                 ← 所有密鑰設定
│   ├── requirements.txt     ← Python 套件清單
│   └── ngrok.exe            ← HTTPS 通道工具
│
外部服務
├── LINE Messaging API       ← 接收用戶訊息
├── Anthropic Claude API     ← AI 圖片辨識
├── Google Sheets API        ← 資料備份（雲端同步，非同步）
└── Google Drive API         ← 圖片備份（雲端硬碟）
```

**資料流向：**
```
LINE 用戶傳訊息
    → LINE 平台推送到 Webhook URL
    → ngrok 轉入本機 8001 port
    → FastAPI 處理
    → Claude API 辨識
    → ① 優先寫入 ERP bot.xlsx（本地，毫秒級）
    → ② 非同步同步到 Google Sheets（背景，失敗不影響主流程）
    → 圖片備份到 Google Drive
    → 回覆用戶
```

> **架構說明（2026-07 更新）**：採用「本地優先」策略。`ERP bot.xlsx` 是主要資料來源，Google Sheets 為雲端備份。即使 Sheets API 出現 503 等暫時錯誤，資料仍安全寫入本地 xlsx，不影響使用者體驗。

## 第一次安裝

> 只需要做一次，之後重開機直接看「每次啟動流程」

### 步驟 1：安裝 Python 套件

開啟 PowerShell，切換到專案資料夾：

```powershell
cd "F:\AI\line-sme-bot"
pip install -r requirements.txt
```

> ⚠️ 若系統有多個 Python 版本，請確認用的是 **Python 3.10 系統版本**的 pip，而非 venv：
> ```powershell
> C:\Users\Administrator\AppData\Local\Programs\Python\Python310\Scripts\pip.exe install -r requirements.txt
> ```

安裝完成後確認：
```powershell
pip show fastapi uvicorn anthropic openpyxl
```
四個套件都有出現即代表成功。

> `openpyxl` 是本地 xlsx 功能的必要套件，請確認有出現在清單中。

---

### 步驟 2：設定 .env 檔案

`.env` 在 `F:\AI\line-sme-bot\.env`，確認以下欄位都已填入（詳細說明見[環境變數說明](#環境變數說明env)）：

```
LINE_CHANNEL_ACCESS_TOKEN=...
LINE_CHANNEL_SECRET=...
ANTHROPIC_API_KEY=...
GOOGLE_SERVICE_ACCOUNT_JSON=...
GOOGLE_SPREADSHEET_ID=...
GOOGLE_DRIVE_FOLDER_ID=...
```

---

### 步驟 3：設定 ngrok

1. 至 [dashboard.ngrok.com](https://dashboard.ngrok.com) 登入
2. 左側選單 → **Getting Started → Your Authtoken**
3. 複製 token，執行：

```powershell
cd "F:\AI\line-sme-bot"
.\ngrok.exe config add-authtoken 你的token
```

---

### 步驟 4：設定 Google Service Account

1. 開啟 [Google Cloud Console](https://console.cloud.google.com)
2. IAM → Service Accounts → 建立並下載 JSON 金鑰
3. 將 JSON 內容（壓成一行）填入 `.env` 的 `GOOGLE_SERVICE_ACCOUNT_JSON`
4. 在 Google Sheets → 共用 → 加入 Service Account email（編輯者）
5. 在 Google Shared Drive → 管理成員 → 加入 Service Account email（內容管理員）

---

### 步驟 5：設定 LINE Webhook

1. 先啟動伺服器和 ngrok（見下一節）
2. 取得 ngrok 網址，例如：`https://xxxx.ngrok-free.app`
3. 至 [LINE Developers Console](https://developers.line.biz/console/)
4. 選擇你的 Channel → Messaging API → Webhook URL
5. 填入：`https://xxxx.ngrok-free.app/webhook`
6. 點「Verify」出現 **Success** 即完成

## 每次啟動流程

每次重開機或要使用系統時，需依序執行以下步驟。

### 步驟 A：啟動 Bot 伺服器

開啟第一個 PowerShell 視窗：

```powershell
cd "F:\AI\line-sme-bot"
uvicorn app.main:app --host 0.0.0.0 --port 8001 --reload
```

看到以下訊息代表成功：
```
INFO:     Application startup complete.
INFO:     Uvicorn running on http://0.0.0.0:8001
```

> **不要關閉這個視窗**，關閉後系統停止運作

---

### 步驟 B：啟動 ngrok 通道

開啟**第二個** PowerShell 視窗：

```powershell
cd "F:\AI\line-sme-bot"
.\ngrok.exe http 8001
```

看到以下畫面代表成功：
```
Forwarding  https://xxxx-xxxx.ngrok-free.app -> http://localhost:8001
```

記下這個 `https://xxxx.ngrok-free.app` 網址。

> **不要關閉這個視窗**，關閉後 LINE 無法連線

---

### 步驟 C：更新 LINE Webhook URL（若網址變了）

ngrok 免費版每次重開網址會改變。若網址更新了：

1. 複製新的 ngrok 網址
2. 至 LINE Developers Console → Webhook URL
3. 更新為：`https://新網址/webhook`
4. 點「Verify」確認成功

> 💡 購買 ngrok 付費方案可以固定網址，就不需要每次更新

---

### 步驟 D：開啟 Dashboard（選用）

瀏覽器開啟：
```
http://localhost:8001/dashboard
```

---

### 快速確認清單

每次啟動後確認以下三點：

```
□ 瀏覽器能開啟 http://localhost:8001/health
  （應顯示 {"ok":true,"model":"claude-sonnet-4-6"}）

□ ngrok 視窗顯示 Forwarding https://...

□ LINE Developers Console Webhook URL 與 ngrok 網址一致
```

## 環境變數說明（.env）

設定檔位置：`F:\AI\line-sme-bot\.env`

```dotenv
# ─── 本地資料庫 ─────────────────────────────────────────
XLSX_PATH=F:\AI\line-sme-bot\ERP bot.xlsx
# 本地 xlsx 主要資料庫路徑，預設為專案根目錄下的「ERP bot.xlsx」
# 本機開發直接用此路徑；Railway 部署時此檔案不持久化（請改用 Sheets 或 DB）

# ─── LINE ──────────────────────────────────────────────
LINE_CHANNEL_ACCESS_TOKEN=...
# 從 LINE Developers → Channel → Messaging API → Channel access token 複製

LINE_CHANNEL_SECRET=...
# 從 LINE Developers → Channel → Basic settings → Channel secret 複製

# ─── AI 辨識 ───────────────────────────────────────────
ANTHROPIC_API_KEY=sk-ant-...
# 從 https://console.anthropic.com/ → API Keys 取得

CLAUDE_MODEL=claude-sonnet-4-6
# 固定值，不需修改

# ─── Google ────────────────────────────────────────────
GOOGLE_SERVICE_ACCOUNT_JSON={"type":"service_account",...}
# Service Account 的 JSON 金鑰，壓成單行

GOOGLE_SPREADSHEET_ID=1ElOac...
# 從 Google Sheets 網址取得：
# https://docs.google.com/spreadsheets/d/【這裡】/edit

GOOGLE_DRIVE_FOLDER_ID=1fvA6M...
# 從 Google Shared Drive 網址取得：
# https://drive.google.com/drive/folders/【這裡】

# ─── 警示設定 ──────────────────────────────────────────
AMOUNT_ALERT_THRESHOLD=100000
# 金額超過此值時通知管理員（單位：台幣）

ADMIN_LINE_USER_ID=Uxxxxxxxx...
# 管理員的 LINE User ID，從 LINE 開發者後台取得
# 留空則不發警示

# ─── 其他 ──────────────────────────────────────────────
BOT_TIMEZONE=Asia/Taipei
# 時區，台灣固定此值

PUBLIC_BASE_URL=https://xxxx.ngrok-free.app
# 目前的 ngrok 網址（每次重開 ngrok 後需更新）
```

### 健康檢查

```powershell
curl http://localhost:8001/health
```

預期回應：
```json
{"ok": true, "model": "claude-sonnet-4-6"}
```

### Dashboard 確認

### 手動測試 OCR

使用 PowerShell 傳送測試圖片（需要有圖片檔案）：

```powershell
# 將圖片轉 base64
$bytes = [System.IO.File]::ReadAllBytes("C:\path\to\receipt.jpg")
$b64 = [Convert]::ToBase64String($bytes)

# 呼叫測試 API
$body = @{image_b64=$b64; media_type="image/jpeg"} | ConvertTo-Json
Invoke-RestMethod -Uri "http://localhost:8001/admin/test-ocr" `
  -Method POST -Body $body -ContentType "application/json"
```

## 查看日誌

Bot 伺服器啟動後，所有訊息會即時顯示在 PowerShell 視窗中。

### 日誌格式

```
2026-05-10 14:32:01 INFO  sme-bot: text intent=accounting user=U123456
2026-05-10 14:32:02 INFO  sheets_service: appended to 記帳流水帳: 記帳流水帳!A28:G28
2026-05-10 14:33:15 INFO  image_handler: OCR completed confidence=高
2026-05-10 14:33:17 INFO  drive_service: uploaded 20260510_AB-12345678_發票.jpg
```

### 常見日誌說明

| 訊息 | 意思 |
|------|------|
| `text intent=accounting` | 文字記帳觸發 |
| `text intent=order` | 叫貨訂單觸發 |
| `text intent=unknown` | 無法辨識，交給 AI 判斷 |
| `OCR completed confidence=高` | 圖片辨識成功 |
| `appended to 收據記錄` | 寫入 Sheets 成功 |
| `uploaded 20260510_...` | 上傳 Drive 成功 |

### 查看錯誤

日誌中 `ERROR` 開頭的訊息表示有問題，例如：

```
2026-05-10 14:35:00 ERROR sheets_service: append_receipt error: ...
```

## 更新程式碼

若程式有更新（例如修改了 `.py` 檔案）：

**開發模式下（--reload）自動重載**，不需要重新啟動，儲存檔案後 uvicorn 會自動偵測並重啟。

若需要完全重新啟動：

1. 在 Bot 伺服器的 PowerShell 視窗按 `Ctrl + C` 停止
2. 重新執行啟動指令：
   ```powershell
   uvicorn app.main:app --host 0.0.0.0 --port 8001 --reload
   ```

## 長期部署（Railway）

若不想每次開電腦都要手動啟動，可以部署到 Railway 雲端，24 小時持續運行。

### 部署步驟

1. 至 [railway.app](https://railway.app) 建立帳號
2. 新增專案 → Deploy from GitHub repo（先把程式推到 GitHub）
3. 設定環境變數（把 `.env` 的內容逐一填入 Railway 的 Variables）
4. 部署完成後取得固定網址（例如：`https://sme-bot.up.railway.app`）
5. 更新 LINE Webhook URL 為 Railway 網址

### Railway 環境變數設定

在 Railway 後台 → Variables → Add all variables：

```
LINE_CHANNEL_ACCESS_TOKEN
LINE_CHANNEL_SECRET
ANTHROPIC_API_KEY
GOOGLE_SERVICE_ACCOUNT_JSON
GOOGLE_SPREADSHEET_ID
GOOGLE_DRIVE_FOLDER_ID
AMOUNT_ALERT_THRESHOLD
ADMIN_LINE_USER_ID
BOT_TIMEZONE
```

`PUBLIC_BASE_URL` 填入 Railway 給的固定網址（不是 ngrok）。

## 常見錯誤處理

### 錯誤：`ModuleNotFoundError: No module named 'xxx'`

**原因：** 套件未安裝  
**解法：**
```powershell
cd "F:\AI\line-sme-bot"
pip install -r requirements.txt
```

> ⚠️ **注意 Python 環境**：Bot 使用的是系統 Python（`C:\Users\Administrator\AppData\Local\Programs\Python\Python310`），不是 `.venv`。確認用的是**系統 pip**，不是 venv pip：
> ```powershell
> C:\Users\Administrator\AppData\Local\Programs\Python\Python310\Scripts\pip.exe install -r requirements.txt
> ```

---

### 錯誤：`No module named 'openpyxl'`

**原因：** openpyxl 未安裝到系統 Python（本地 xlsx 功能需要此套件）  
**解法：** 雙擊執行專案根目錄的 `install_openpyxl.bat`，或執行：
```powershell
C:\Users\Administrator\AppData\Local\Programs\Python\Python310\Scripts\pip.exe install openpyxl>=3.1.0
```

---

### 錯誤：`ValidationError` 或 `.env` 讀取失敗

**原因：** `.env` 有欄位空白或格式錯誤  
**解法：** 確認 `.env` 中所有必填欄位都有值，特別是：
- `LINE_CHANNEL_ACCESS_TOKEN`
- `ANTHROPIC_API_KEY`
- `GOOGLE_SERVICE_ACCOUNT_JSON`（必須是完整的 JSON 單行字串）

---

### 錯誤：LINE Webhook Verify 失敗

**原因可能是：**
1. 伺服器沒有在跑 → 確認 PowerShell 視窗還開著
2. ngrok 沒有在跑 → 確認第二個 PowerShell 視窗
3. Webhook URL 填錯 → 確認結尾是 `/webhook`，不是 `/`
4. `LINE_CHANNEL_SECRET` 填錯 → 回 LINE Developers Console 確認

---

### 錯誤：Google Sheets 寫入失敗（403）

**原因：** Service Account 沒有試算表的編輯權限  
**解法：**
1. 開啟 Google Sheets
2. 右上角「共用」
3. 加入 Service Account email（格式：`xxx@xxx.iam.gserviceaccount.com`）
4. 選「編輯者」

---

### 錯誤：Google Drive 上傳失敗（storageQuotaExceeded）

**原因：** Service Account 無法上傳到個人 Google Drive  
**解法：** 使用「共用雲端硬碟（Shared Drive）」
1. Google Drive 左側欄 → 共用雲端硬碟 → 新增
2. 加入 Service Account 為「內容管理員」
3. 將 Shared Drive 的 ID 填入 `.env` 的 `GOOGLE_DRIVE_FOLDER_ID`

---

### 錯誤：ngrok `ERR_NGROK_4018`（需要驗證帳號）

**解法：**
1. 至 [dashboard.ngrok.com](https://dashboard.ngrok.com) 註冊並登入
2. 複製 Authtoken
3. 執行：`.\ngrok.exe config add-authtoken 你的token`

---

### 錯誤：Port 8001 已被使用

```
ERROR: [Errno 10048] Only one usage of each socket address
```

**解法：** 找出並結束佔用 port 的程式：
```powershell
netstat -ano | findstr :8001
taskkill /PID 找到的PID /F
```

---

### 系統完全無回應排查順序

```
1. 確認 uvicorn 視窗是否還在執行
2. 瀏覽器開 http://localhost:8001/health 是否有回應
3. 確認 ngrok 視窗是否還在執行
4. 確認 LINE Developers Console Webhook URL 是否正確
5. 執行 python test_integration.py 確認 Google API 連線
```

---

*如有其他問題，請查看 uvicorn 視窗中的 ERROR 訊息*
